Key Design Decisions
Decision	Rationale
XML-first parsing with regex fallback	Small LLMs often produce malformed XML; dual strategy maximizes extraction success
HandoffPacket
 as the inter-model message	Clean data boundary — models never touch each other's raw output
SHA-256 cache keying	Deterministic, collision-resistant, supports profile-scoped keys
Separate 
generate_demonstrations()
 / 
generate_final()
Enables N-model chaining beyond the default 2-model pipeline
ollama.Client() over global functions	Supports custom hosts / multi-server setups
Bug Fix During Verification
The 
_try_xml_parse
 function in 
parser.py
 was using ET.tostring() which includes element tail text (text after the closing tag). This caused 
parse_answer()
 to return surrounding text alongside the answer. Fixed by reconstructing inner content from elem.text and explicit child serialization.
